观察题目内容

  • 定义静态数组result
    因为题目返回格式为int*，所以返回数组首地址。由于传入的参数中并没有这个数组，所以需要自己定义。

由于是在自定义函数中进行定义的，且需要将定义的这个变量返回主函数，所以采用static

static的用法:
存储在静态数据区的变量会在程序刚开始运行时就完成初始化，也是唯一的一次初始化。共有两种变量存储在静态存储区：全局变量和 static 变量，只不过和全局变量比起来，static 可以控制变量的可见范围，说到底 static 还是用来隐藏的。

  • 算法部分
    外层for中i从0到numSize,内层for中j从i+1到numSize。因为num[1]+num[2]=6与num[2]+num[1]=6是一样的结果，没有必要再计算一遍。所以i，j并没有单独遍历完整个数组。
    [注：对于少量数据可能会执行得比下面这种方式快，但是对于大量的数据来说，这就很微不足道，时杂还是o(n²)没有变]

    当我们使用遍历整个数组的方式寻找 target - x 时，需要注意到每一个位于 x 之前的元素都已经和 x 匹配过，因此不需要再进行匹配。而每一个元素不能被使用两次，所以我们只需要在 x 后面的元素中寻找 target - x。

哈希表

思想：

对比暴力枚举法，枚举数组中的每个数x。根据具体的x依次遍历寻找数组中是否存在target-x[这部分查找o(n)]。

而哈希表则是在枚举数组中的每个数x时，计算target-x，再查找哈希表对应中是否存在对应的value[这部分查找o(1)]。

创建一个哈希表，对于每一个 x，我们首先查询哈希表中是否存在 target - x，然后将 x 插入到哈希表中，即可保证不会让 x 和自己匹配。
